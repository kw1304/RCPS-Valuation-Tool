# 감사전 리스크 확인 툴 (RISK_TOOL)

감사계획 단계에서 대상기업의 중요왜곡위험을 **DART 자동수집** 재무·공시 + 뉴스로
4축 룰베이스 신호등 스크리닝하고 **ISA 315 위험평가 조서**를 생성한다.

- 서비스: FastAPI, `http://127.0.0.1:8533` (WAT 셸 `risk-precheck`로 임베드)
- 기동: `python run_server.py` (또는 부팅 자동시작 `deploy/RISK_Server.vbs`)
- 테스트: `python -m pytest` (conftest가 `src`를 path에 추가)

## 리스크 4축 (신호등 green/yellow/red/na)

1. **분석적검토 (ISA 520)** — 10개 지표: 매출증감·매출총이익률·영업이익률·판관비율·
   매출채권회전율·재고회전율·순이익률·영업CF/매출·총자산회전율·매입채무회전율.
   모든 신호는 **수행중요성(PM) 이중게이트**(% 밴드 AND 변동금액>PM)로 오탐 차단.
2. **부정 (ISA 240)** — 발생주의 조작(흑자&영업CF음수)·가공매출(매출채권갭)·과대재고(재고갭)·
   유효세율 이상. 갭 신호는 관련잔액이 PM 초과일 때만 발화.
3. **계속기업 (ISA 570)** — 부채비율·이자보상배율(3년연속<1 한계기업)·유동비율·영업CF·자본잠식.
4. **외부 정성** — 네이버 뉴스(회사명 제목매칭) + DART 공시이벤트. AI가 구조화
   (사건유형·날짜·요약·재무영향·출처). 신호등 아님(정성 참고).

**종합등급**: 계속기업 적신호 또는 중대부정(발생주의조작/가공매출/과대재고) 단독 → 높음.
분석적 변동만은 적신호 5개↑일 때만 높음. (변별력 위해 단순 적신호 1개로 높음 안 매김.)

## 수행중요성(PM)
- **자동**: 세전이익5%/매출0.5%/자산0.5% 중 가장 작은(보수적) 채택 × 75%.
- **직접 지정**: 기준(매출/자산/세전이익/자본) + 중요성% + 수행% 입력.

## 데이터 커버리지
- 상장사·사업보고서 제출 비상장: DART 구조화 API (CFS 연결 우선).
- 순수 외감 비상장: 감사보고서 원문(document.xml) 표 파싱 fallback.
- **금융·보험업**(KSIC 64/65/66): 부채비율·유동비율 부적용(na) — 예수금·보험부채로 구조 상이.
- 회사별 account_id 변형에 강건(`_FIELD_IDS` 우선순위 다중매핑). 결측은 None 유지(조용히 0 금지).

## 데이터 품질 검증 (자동 경고)
- 자산 = 부채 + 자본 항등식, 매출/자산 스케일, 핵심계정 결측을 런타임 점검 →
  대시보드 경고배너 + Excel 표지. 미검증 회사의 추출오류를 감사인에게 surfacing.

## AI (선택)
- 로그인된 **Claude CLI** 사용(API 키 불필요). 없으면 신호·뉴스만 degrade.
- 신호 코멘트 + 뉴스/공시 구조화를 **1회 호출로 통합**(claude CLI 동시실행 불가 회피).
  분석 1회 약 30초.

## 산출물
- 화면 대시보드: 종합등급·4축 신호카드·AI 위험이벤트·뉴스·공시·데이터품질 경고.
- Excel 조서(6시트): 표지·재무요약·위험평가매트릭스·신호상세·외부리스크·후속감사절차.
