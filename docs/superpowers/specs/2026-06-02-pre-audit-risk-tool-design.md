# 감사전 리스크 확인 툴 (Pre-Audit Risk Tool) — 설계

작성일: 2026-06-02
상태: 설계 승인 완료 (구현 계획 대기)

## 1. 목적

감사계획 단계(fieldwork 이전)에서 대상 기업의 중요왜곡위험을 자동 스크리닝한다.
DART OPEN API로 재무·공시를 자동 수집하고, 룰베이스 임계값으로 신호등을 산출하며,
뉴스 리서치로 외부 정성리스크를 파악해 통합 리스크 대시보드 + ISA 315 위험평가 감사조서를 생성한다.

감사기준 정합: ISA 315(위험평가), ISA 520(분석적절차), ISA 240(부정), ISA 570(계속기업).

## 2. 범위 결정 (브레인스토밍 확정)

| 항목 | 결정 |
|---|---|
| 리스크 축 | 통합 4축 (이상징후 + 부정 + 계속기업 + 외부정성) |
| 데이터 수집 | DART API 자동 — 상장(구조화API) + 비상장 외감(감사보고서 PDF 파싱) |
| 산출 방식 | 룰베이스 신호등(방어가능·재현성) + AI 코멘트(보조) |
| 외부 리스크 | 자동 뉴스 리서치(WebSearch) + DART 공시이벤트 |
| 산출물 | 화면 대시보드 + Excel 감사조서(내보내기) |
| 통합 | 독립 디렉터리 `RISK_TOOL/` FastAPI 서비스 + WAT 임베드 셸 UI |

## 3. 아키텍처

AUTO_JET 클린아키텍처 패턴 답습. WAT 셸 표준(헤더 padding-left 7.25rem, 푸터 통일 문구) 적용.

```
RISK_TOOL/
  src/
    domain/             # 위험지표·임계값·신호등 규칙 (순수 로직 — 테스트 핵심)
      indicators.py     #   비율·증감 계산
      thresholds.py     #   임계값 상수 + 신호등 판정
      materiality.py    #   수행중요성(PM) 산정
      risk_grade.py     #   종합 위험등급 집계
    application/
      assess_risk_uc.py # 수집→평가→코멘트 오케스트레이션
    infrastructure/
      dart/             # rcps dart_financials + CC client 이식
      news/             # WebSearch 리서치 + 키워드 스캔
      llm/              # AI 코멘트·뉴스요약 (Claude API)
      excel/            # ISA 315 위험평가 W/P 생성
    interface/
      api/              # FastAPI 라우트
        frontend/       # WAT 셸 대시보드
  tests/
    unit/               # domain 임계값 경계 테스트
    integration/        # DART fixture 회귀
```

### 단위별 책임 (isolation)
- **domain**: 외부 의존 0. 재무딕트 입력 → 신호·등급 출력. 결정론적·골든값 테스트.
- **infrastructure/dart**: 회사명→재무 5개년·공시리스트. 기존 코드 재활용.
- **infrastructure/news+llm**: 비결정적 외부. application에서 try/degrade로 격리.
- **application**: 순서 조립만. 비즈니스 임계값은 domain에 위임.

## 4. 데이터 수집 (DART 자동)

1. 회사명 → corp_code (CC `client.py` 재활용)
2. 재무 5개년 (rcps `dart_financials.py` — 상장 구조화API / 비상장 외감 감사보고서 document.xml 파싱 fallback)
3. 공시 리스트 최근 1년 — 주요사항보고서: 감자·소송·횡령배임·채무보증·부도·영업양수도

## 5. 수행중요성(PM) 자동산정

- benchmark: 세전계속영업이익 5% / 매출액 0.5% / 자산총계 0.5% 중 적용 (비상장 외감 실무 흔히 매출·자산 0.5%)
- 수행중요성 PM = 중요성 × 75% (기본; 50~75% 범위)
- 축1 분석적검토 신호의 **이중게이트** 분모로 사용 (오탐 차단)

## 6. 리스크 4축 — 룰베이스 임계값

신호등: 녹(정상) / 황(주의) / 적(유의).

### 축1. 분석적검토 (ISA 520)
모든 신호 = **% 밴드 충족 AND |변동금액| > PM** 동시. 한쪽만이면 '관찰'(신호 아님).

| 지표 | 황 | 적 |
|---|---|---|
| 매출 증감률 | ±10% | ±30% |
| 매출총이익률(GPM) 변동 | ±2%p | ±5%p |
| 영업이익률(OPM) 변동 | ±2%p | ±5%p |
| 매출채권회전율(DSO 악화) | -20% | -35% |
| 재고회전율 하락 | -20% | -35% |
| 판관비율(SG&A/매출) 변동 | ±2%p | ±5%p |

### 축2. 부정 (ISA 240)
| 지표 | 황 | 적 |
|---|---|---|
| 순이익 흑자 & 영업CF 음수 | — | 적 (발생주의 조작) |
| 매출채권증가율 − 매출증가율 | >10%p | >25%p (가공매출) |
| 재고증가율 − 매출증가율 | >15%p | >30%p (과대재고) |
| 유효세율 | <10% or >35% | 음수 or >50% |
| 특수관계자거래 비중 | 데이터 있으면 표기 | — |

### 축3. 계속기업 (ISA 570)
| 지표 | 황 | 적 |
|---|---|---|
| 부채비율 | >200% | >400% or 자본잠식 |
| 이자보상배율 | <1 | 3년 연속<1(한계기업/좀비) or <0 |
| 유동비율 | <100% | <50% |
| 영업CF | 음수 1회 | 연속 음수 |
| 자본잠식 | 부분 | 완전(자본총계<0) |

### 축4. 외부 정성 (뉴스·공시) — AI 보조, 신호등 아님
- WebSearch: `회사명 + 리스크키워드`. 키워드셋: 소송/횡령/배임/분식회계/적자전환/자본잠식/감자/부도/영업정지/대표이사 변경/회생/워크아웃/관리종목/상장폐지/세무조사/리콜
- DART 주요사항보고서 최근 1년 매칭
- AI(Claude): 뉴스+공시 → 구조화(사건유형·날짜·요약·재무영향·출처URL). 결과 없으면 "특이사항 없음" 명시.
- 동명이인·계열사 혼동 방지: AI에 업종·소재지 식별자 제공해 필터. 무근거 추측 금지.

### 종합 위험등급 (결정론적 룰)
- 적신호 ≥1 또는 황신호 ≥3 → **높음**
- 황신호 1~2 → **보통**
- 그 외 → **낮음**

## 7. AI 코멘트 (룰신호 보조)
- 각 축 룰 신호 → AI가 "왜 위험·후속 확인사항·관련 경영진주장(assertion)" 서술.
- 신호 자체는 절대 불변(룰베이스 고정). AI는 등급에 영향 없음.
- 프롬프트: K-IFRS·감사기준 인용, 한국 회계용어(장부가 등) 준수.

## 8. Excel 감사조서 — ISA 315 위험평가 W/P
1. **표지**: 회사·기준연도·PM 산정근거·종합위험등급
2. **재무요약**: 5개년 주요계정·비율 추세
3. **위험평가 매트릭스**: 계정 × 경영진주장(실재성·완전성·평가·권리의무) → 식별위험·신호근거·평가(고/중/저)
4. **4축 신호상세**: 지표·실측값·임계값·신호등·AI코멘트
5. **외부리스크**: 뉴스·공시 카드(사건·날짜·요약·출처)
6. **후속감사절차 제안**: 신호별 권고절차 (예: 매출채권회전율 하락 → 채권조회·기수령 검토)

## 9. 화면 대시보드
WAT 셸. 회사명 입력 → 진행상태 → 결과: 종합등급 신호등 + 4축 카드(추세 미니차트) + 뉴스카드 + "조서 내보내기" 버튼.

## 10. 에러처리
- DART 자료없음(비상장 사업보고서 미제출) → 감사보고서 PDF fallback → 실패 시 **명시적 수기입력 경로** (조용히 0 채우기 금지 — `bc_rowparser_realdata_bugs` 교훈)
- 비율 분모 0/음수 → N/A 표기, 신호 보류
- WebSearch·LLM 실패 → 축4만 degrade, 축1~3 정상 진행
- DART 키 환경변수, 캐시 `~/.risk_tool/dart_cache`

## 11. 테스트
- **domain 단위테스트(핵심)**: 각 지표 임계값 경계(황/적 ±경계), PM 게이트 on/off, 분모0, 자본잠식, 3년연속 판정. 결정론적 골든값.
- **통합**: DART 응답 fixture(상장 1·비상장외감 1) → 4축 산출 회귀
- **뉴스/LLM**: mock으로 외부의존 격리

## 12. 비범위 (YAGNI)
- 후속측정·attribution·분기비교 없음 (`feedback_no_subsequent`)
- 분개특성 상세 테스트는 AUTO_JET 영역 — 여기선 재무비율 레벨만
- 화면에 "5803" 등 baseline 식별자 노출 없음

## 13. 재사용 자산
- `rcps_valuation/inputs/dart_financials.py` — 상장+비상장 외감 5개년 (DartFinancials)
- `CC_SAMPLING_TOOL_V2/src/infrastructure/dart/client.py` — corpCode·공시 list·document.xml
- WAT 임베드 셸 표준 (`wat_tool_standard`)
- AUTO_JET 클린아키 레이어 패턴
